Restriction

The Restriction API can be used to manage the restriction data on the server.

Method

Request URL

Description

GET

api/v7/Restriction

Retrieves information on the specified restriction data.

POST

api/v7/Restriction

Adds a new set of restriction data

PUT

api/v7/Restriction

Adds or updates a restriction data.

GET

api/v7/Restriction/{restrictionCode}

Retrieves information on a specified restriction data using its code.

HTTP request details

GET api/v7/Restriction
Request information

None.

Response information

The API response indicates the success or failure status of the GET api/v7/Restriction request. The following table describes the information contained in the API response.

Name

Description

RestrictionCode

Specifies the code of the restriction data.

startDate

Specifies the start date.

endDate

Specifies the end date.

startDay

Specifies the first day.

startHour

Specifies the start hour.

startMinute

Specifies the start minute.

endDay

Specifies the end day.

endHour

Specifies the end hour.

endMinute

Specifies the end minute.

customerAuditDate

Defines the audit information.

culture

Specifies the culture code of the restriction data.

Example: The following are sample values for this parameter.

•en-US

•fr

•fr-CA

•es

•ja-JP

description

Defines the text description of the age code.

Note: This parameter is culture-specific.
