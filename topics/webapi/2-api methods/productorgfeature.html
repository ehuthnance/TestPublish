ProductOrgFeature

The ProductOrgFeature API can be used to manage a product's features, such as warranty, pre-order properties, and required fees.

Method

Request URL

Description

GET

api/v7/ProductOrgFeature

Retrieves the feature data of all the products in all stores.

POST

api/v7/ProductOrgFeature

Adds new feature data to a product in a specific store.

PUT

api/v7/ProductOrgFeature

Updates the feature data of a product in a specific store.

GET

api/v7/ProductOrgFeature/{ProductCode}?OrgName={OrgName}

Retrieves the feature data of a product using the product code and store number.

HTTP request details

GET api/v7/ProductOrgFeature
Request information

None.

Response information

The API response indicates the success or failure of the GET api/v7/ProductOrgFeature request.

Name

Description

orgName

Specifies the store number of the product.

ProductCode

Specifies the code of the product.

featureKey

Specifies the key that identifies the product feature, such as:

•WarrantyAddOn—Use this feature key to indicate that the item is a warranty.

•WarrantyEligible—Use this feature key to indicate that the item is warranty–eligible.

•PreOrder—Use this feature key to indicate that the product is a pre-order item.

•RequiredFeeItem—Use this feature key to indicate that the product is an item with a required fee.

featureData

Defines the feature data.

Note: The feature data options vary depending on the feature key. The JSON object for featureData should also be properly formatted.

The following describes the feature data for each feature key.

 

WarrantyAddOn

•Criteria—The condition on the warranty that indicates the price range of the warranty items.

•Term—The value used to calculate the warranty end date and determines the term validity of the warranty item. The terms can be configured in months and years. When the term is not configured, the warranty end date will not be calculated.

•TCFilename—The file name of the terms and conditions of the warranty items. The terms and conditions text file is added in %ProgramData%\NCR\AdvancedStore\POS. When the terms and conditions text file is not configured, a default text will be displayed on the PIN pad.

•RequireCCToken—A flag that determines whether to capture the credit card token.

•True—Prompts for the card token.

•False—Does not prompt for the card token.

 

WarrantyEligible

•WarranyAddOnGroup—The value used to indicate that the the available warranty items can be grouped by category.

 

PreOrder

•startDate—The date at which the product becomes a pre-order item.

•endDate—The date at which the product stops being a pre-oder item.

•maximumQuantity—The order limit number of the pre-order item per transaction.

•depositAmount—The deposit amount of the pre-order item. The value for this option should be an integer.

 

RequiredFeeItem

•startDate—The date at which the required fee of the item becomes active.

•endDate—The date at which the required fee of the item becomes null.

•productCode—The code of the required fee of the item.

  {
    "orgName": "111",
    "ProductCode": "12345678901234",
    "featureKey": "PreOrder",
    "featureData": {"startDate':'2020-01-01T00:00:00', 'endDate':'2020-01-05T23:59:59', 'maximumQuantity': '2', 'depositAmount':'10"}
  },

  }
]
