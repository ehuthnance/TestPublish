Encryption error, POS shuts down

When launching the POS for the first time, application shuts down due to an encryption error.

Use the following information to determine the problem’s possible cause and solution.

Possible causes

The encryption error may be caused by any of the following factors:

Encryption files missing from the Advanced Store POS directory

When launching the POS for the first time, the following encryption files are created in the %ProgramData%\NCR\AdvancedStore\POS directory.

•<xxxxxxx>.lck

•ENCFILE

Check if these encryption files are present in the AS POS directory. If the POS failed to create these encryption files, the POS displays an encryption error and shuts down.

Solution

Re-launch the POS application. The POS should automatically create the files in the folder and start properly.

Windows Firewall is blocking the Advanced Store POS application

The POS application and corresponding programs are blocked by the Windows Firewall.

Solution

Enable the POS application and its corresponding programs through the Windows Firewall.
