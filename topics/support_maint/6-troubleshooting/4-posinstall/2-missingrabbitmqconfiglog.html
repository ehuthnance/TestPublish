RabbitMQ config and log files not found or not in correct location

The config and log files required when running the NCR RabbitMQ Setup application are not found in the specified locations in the local drive.

Use the following information to determine the problem’s possible cause and solution.

Possible cause

Environment variables were created after the RabbitMQ service application was installed. NCR recommends creating the RabbitMQ environment variables before installing the RabbitMQ service application. If this order was not applied and the environment variables were created after the RabbitMQ service application was installed, problems may occur in proceeding with the POS installation.

Solution

Uninstall and re-install the RabbitMQ service application. Follow these steps:

1.From the Windows taskbar, select Start→All Programs→RabbitMQ Server. The RabbitMQ Service batch files are displayed.

2.Select the following batch files, one at a time, in this order:

a.RabbitMQ Service—stop

b.RabbitMQ Service—remove

c.RabbitMQ Service—(re)install

d.RabbitMQ Service—start

When a batch file is selected, for example, RabbitMQ Service – stop, a Command Prompt window is displayed. After the command line runs the batch file, the Command Prompt window automatically closes.
