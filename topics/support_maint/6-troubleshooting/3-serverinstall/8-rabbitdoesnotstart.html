RabbitMQ service does not start

The RabbitMQ service does not start during Messaging Server or In-Store Server installation.

Possible causes

This event may be caused by any of the following factors:

Un-synchronized erlang cookies

Verify that the .erlang.cookie file in each of the following directories contain the same value. If not, apply the corresponding solution.

•Windows system directory (%USERPROFILE%)

Example:  C:\Windows\system32\config\systemprofile

•User’s home directory (%HOMEDRIVE%%HOMEPATH%)

Example:  C:\Users\<install user>

Solution

If the files do not have the same value, synchronize the erlang.cookie files. Copy the .erlang.cookie file from the Windows directory to the User's directory, replacing the cookie file that was already there.

RabbitMQ service was not properly installed

The RabbitMQ service is installed to the system when running the NCR RabbitMQ Setup application. If the service does not start and the .erlang.cookies are already synchronized, the problem may be due to the service not being installed properly by the setup application.

Solution

Re-install the RabbitMQ service using the following methods:

Re-install service using RabbitMQ batch files

To re-install the service using the RabbitMQ batch files, follow these steps:

1.Access the Windows apps list screen. The RabbitMQ Service batch files are displayed.

2.Select the following batch files, one at a time, in this order:

a.RabbitMQ Service—stop

b.RabbitMQ Service—remove

c.RabbitMQ Service—(re)install

d.RabbitMQ Service—start

When a batch file is selected, for example, RabbitMQ Service – stop, a Command Prompt window is displayed. After the command line runs the batch file, the Command Prompt window automatically closes.

3.Verify that the RabbitMQ service has been installed and successfully started in Windows Services.

Re-install service using RabbitMQ control commands

To re-install the service using RabbitMQ control commands, follow these steps:

1.Open the RabbitMQ Command Prompt window.

2.Run the following commands, one at a time, in this order:

a.rabbitmq-service remove

b.rbbitmq-service install

c.rabbitmq-service start

3.Verify that the RabbitMQ service has been installed and successfully started in Windows Services.

Reinstall service using Command Prompt window

To re-install the service using Command Prompt window, follow these steps:

1.At the Command Prompt window, go to the RabbitMQ sbin folder.

Example:  %ProgramFiles%\RabbitMQ Server\rabbitmq_server-n.n.n\sbin

2.Run the following commands, one at a time, in this order:

a.rabbitmq-service remove

b.rabbitmq-service install

c.rabbitmq-service start

3.Verify that the RabbitMQ service has been installed and successfully started in Windows Services.

Data remaining from previous RabbitMQ Server application

Old data may have remained in the system after uninstalling a previous version of the RabbitMQ Server application. Old data will affect the new RabbitMQ Server application installation and may prevent the RabbitMQ service from starting.

Solution

Re-install Erlang and the RabbitMQ Server application.

Warning: This task will remove existing store queues and taxonomy from the system. Make sure to back up these specified data to restore them after the re-installation. Information about backing up and restoring data is provided on the RabbitMQ website (www.rabbitmq.com).

Follow these steps:

1.Uninstall Erlang and the RabbitMQ Server application.

2.Delete the following folders and files from their specified directories:

Folder or file name

Directory

erl<#.#> folder

%ProgramFiles%

RabbitMQ Server folder

%ProgramFiles%

RabbitMQ folder

The RabbitMQ folder may be in one of the following directories:

•in the directory specified in the RabbitMQ_Base environment variable

•in the user's AppData directory

Example: C:\Users\<install user>\AppData\Roaming

.erlang.cookie files

Note: The .erlang.cookie file in each directory must be deleted.

 

•Windows system directory (%USERPROFILE%)

Example: C:\Windows\system32\config\systemprofile

•User’s home directory (%HOMEDRIVE%%HOMEPATH%)

Example: C:\Users\<install user>

3.Open Registry editor, and then, delete the RabbitMQ registry key from the following directory:
HKEY_LOCAL_MACHINE→SOFTWARE→Ericsson→Erlang→ErlSrv→1.1.

4.Restart the system.

5.After the system restarts, re-install Erlang and the RabbitMQ Server application.

6.Restart the system.

7.After the system restarts, re-run the NCR RabbitMQ Setup application to install the RabbitMQ service. For more information, refer to the following sections:

•Configuring RabbitMQ in Messaging Server

•Configuring RabbitMQ in In-Store Server

8.Verify that the RabbitMQ service has been installed and successfully started in Windows Services.
