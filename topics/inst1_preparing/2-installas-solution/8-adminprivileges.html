System administrative privileges

A system Administrator is the person responsible for setting up and managing the AS solution. The administrator is responsible for the security and performance of the system and for ensuring that the system performs at its maximum potential. The administrator also manages the system to meet the requirements of its users and thus may install or upgrade applications as necessary.

To perform these management functions and to use the Windows Administrative tools, an administrator must be logged on as a member of the administrator local group of the computer or domain, respectively. Since the administrator can be any member of the administrators group, there can be more than one person with administrator privileges. However, for system management purposes, keep the number of administrators to a minimum.

System administrators are tasked to perform the installation of the Advanced Store solution; otherwise, the user must have administrative privileges.

Note:  All installation packages should be run by right-clicking the file and selecting Run as administrator.
