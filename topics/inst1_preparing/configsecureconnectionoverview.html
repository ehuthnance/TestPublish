Configuring a Secure Connection (HTTPS) when Using DSR Security

The Advanced Store servers can be configured with a secure connection (HTTPS) to encrypt communication between servers. When using DSR Security and HTTPS, this configuration can be applied to allow the EOM and ETS Servers to communicate with the DSR Security API.

To configure a secure connection, additional steps are required during the installation of the servers. The following tasks must be performed on the following servers.

Task

Server

Reference

Configure the web.config file for https.

Web Server

Configuring the Web Server Secure Connection

Configure the EOM DSR Host Protocol and endpoints to use https.

Note: The Remove SSL requirement check box must not be selected when installing the EOM server. For more information, refer to Installing the Customer Configuration Overlay.

EOM Server

Configuring the EOM Secure Connection

Configure the ETS DSR Host Protocol and endpoints to use https.

Note: The Remove SSL requirement check box must not be selected when installing the ETS server. For more information, refer to Installing the Customer Configuration Overlay.

ETS Server

Configuring the ETS Secure Connection

Configure the connection between POS servers and the Enterprise servers to use https.

Note: Ensure that the correct ports are entered when installing the POS. For more information, refer to Connecting the In-Store Server to the Enterprise and Connecting the POS Terminals to the Enterprise.
